## Project Overview

BookVault（破卷书库）is a community-contributed digital library. It has no build step, no framework, and no database — the entire backend is GitHub repository files + a single Cloudflare Pages Function.

## Architecture

Three static HTML pages deployed on Cloudflare Pages, one serverless API endpoint, and GitHub as the storage layer:

```
submit.html → POST /api/submit (CF Function) → GitHub API (PUT books/{cat}/{name}.md)
                                                      ↓
                                            GitHub Actions trigger
                                                      ↓
                                        scripts/generate-index.js
                                                      ↓
                                              public/books.json
                                                      ↓
index.html (reads books.json)    view.html (reads PDF from GitHub raw URLs)
```

- **`public/index.html`** — Book catalog. Fetches `books.json`, renders a filterable/searchable list. Category buttons, client-side search, contributor info masking (email/QQ anonymization).
- **`public/submit.html`** — Submission form. Uploads a PDF (max 50MB), requires Turnstile CAPTCHA, sends as base64 to `/api/submit`. Auto-re-enables submit button after 120s timeout.
- **`public/view.html`** — PDF reader using pdf.js 3.11. Loads PDF from `raw.githubusercontent.com/Guesscode/BookVault/main/` via the `?path=` query parameter. Supports page nav, zoom, keyboard (Left/Right arrows).
- **`functions/api/submit.js`** — The only backend code. Cloudflare Pages Function that handles `POST /api/submit`. Validation pipeline: Turnstile verification → IP rate limiting (3/day via KV with 86400s TTL) → sensitive word regex filtering → field validation → GitHub Contents API commit. Returns CORS headers for browser access.
- **`scripts/generate-index.js`** — Node script that scans `books/{category}/*.md`, extracts metadata from HTML comment headers and markdown titles, writes `public/books.json`. Run by GitHub Actions on every push to `books/**`.
- **`public/_routes.json`** — Cloudflare Pages routing: `/api/*` goes to Functions, everything else is static.

## Book Metadata Format

Each `.md` file in `books/` starts with an HTML comment header generated by the submit function:
```
<!-- 贡献者: {contact} | 提交IP: {ip} | 提交时间: {ISO timestamp} -->
```
The index generator parses this header for `contributor` and `date` fields, and extracts the first `# Title` line.

## Commands

```bash
# Regenerate the book index locally
node scripts/generate-index.js
```

No build, lint, or test commands exist. The site is pure static HTML deployed directly from the `public/` directory.

## Deployment

Deployed on Cloudflare Pages with:
- **Framework preset**: None (no build command)
- **Build output directory**: `public`
- **KV namespace binding**: `RATE_LIMIT` → `BOOKVAULT_RATE_LIMIT` (IP rate limit counters)
- **Environment variables**: `GITHUB_REPO`, `GITHUB_PAT`, `TURNSTILE_SECRET`
- **GitHub Actions**: `.github/workflows/build-index.yml` auto-rebuilds `books.json` on pushes to `books/**`

## Key Constraints

- Book titles max 30 characters, content max 500k chars
- IP rate limit: 3 submissions per 24 hours
- PDF file size limit: 50MB (enforced client-side only)
- Sensitive words are hard-coded as unicode-escaped regex patterns in `functions/api/submit.js`
- Categories are fixed: 数学, 计算机, 物理, 商业, 人文, 历史, 其他 — defined in three places (index.html JS, submit.js validation, generate-index.js scan loop)
- Repository is hard-coded to `Guesscode/BookVault` in four locations (deploy.md, view.html, submit.js, build-index.yml)
- The submit function writes `.md` files to GitHub but the frontend submit page sends PDFs as base64 — there is a mismatch (the submit.js expects `body.content` as text content, but submit.html sends `pdfBase64`). The `body.content` field does not exist in the request payload, which will cause submission failures.
